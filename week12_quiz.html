Quiz: Week 12

1. What is the purpose of using "React Navigation" in a React Native app?

  • A) To handle navigation between screens
  • B) To fetch data from APIs
  • C) To optimize the app's performance
  • D) To manage state in components
Answer: A) To handle navigation between screens
Explanation: React Navigation provides routing and navigation solutions for React Native apps.

2. What is a "Stack Navigator" used for in React Navigation?

  • A) To manage a stack of screens and navigate between them
  • B) To add animations to screens
  • C) To fetch data for each screen
  • D) To implement horizontal scrolling
Answer: A) To manage a stack of screens and navigate between them
Explanation: A Stack Navigator allows users to navigate forward and backward between screens in a stack.

3. What is the "NavigationContainer" component used for?

  • A) To wrap navigators and manage navigation state
  • B) To style the navigation header
  • C) To add transitions between screens
  • D) To connect the app to a backend service
Answer: A) To wrap navigators and manage navigation state
Explanation: The NavigationContainer component is the top-level container for all navigation in a React Native app.

4. What is the difference between "push" and "navigate" in Stack Navigator?

  • A) "push" adds a new screen to the stack, while "navigate" finds an existing screen
  • B) "push" navigates backward, while "navigate" moves forward
  • C) "push" resets the stack, while "navigate" preserves the stack
  • D) There is no difference
Answer: A) "push" adds a new screen to the stack, while "navigate" finds an existing screen
Explanation: The push method adds a new instance of a screen, even if it already exists in the stack, while navigate moves to an existing screen if it exists.

5. What is the purpose of the "headerShown" property in Stack Navigator?

  • A) To toggle the visibility of the navigation header
  • B) To change the color of the header
  • C) To disable back navigation
  • D) To add animations to the header
Answer: A) To toggle the visibility of the navigation header
Explanation: The headerShown property controls whether the navigation header is visible or hidden.

6. What is the purpose of the "initialRouteName" property in Stack Navigator?

  • A) To set the default screen when the app loads
  • B) To navigate to a specific screen
  • C) To disable transitions between screens
  • D) To add headers to all screens
Answer: A) To set the default screen when the app loads
Explanation: The initialRouteName property specifies the first screen to be displayed in the Stack Navigator.

7. What does the "screenOptions" property in Stack Navigator allow you to do?

  • A) Set default options for all screens
  • B) Disable navigation between screens
  • C) Add animations to specific screens
  • D) Load multiple screens at once
Answer: A) Set default options for all screens
Explanation: The screenOptions property is used to define default settings for all screens in the Stack Navigator.

8. What is the purpose of the "Tab Navigator" in React Navigation?

  • A) To switch between screens using tabs
  • B) To manage a stack of screens
  • C) To navigate backward and forward
  • D) To fetch data for different screens
Answer: A) To switch between screens using tabs
Explanation: The Tab Navigator is used to navigate between different screens using a tab-based interface.

  • D) initialRouteName
Answer: A) tabBarOptions

13. What is the purpose of the "lazy" property in Tab Navigator?

  • B) To disable navigation between tabs
  • C) To enable animations for all tabs
  • D) To preload all screens
Answer: A) To load screens only when they are accessed
Explanation: The lazy property ensures that screens in the Tab Navigator are loaded only when they are accessed, improving performance.

  • D) To preload all screens

Explanation: The detachInactiveScreens property detaches screens that are not active, optimizing memory usage and performance.
