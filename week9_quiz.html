Quiz: Week 9 - State Management

1. What is the primary purpose of state in React Native?

  • A) To manage dynamic data within a component
  • B) To store global variables
  • C) To handle API calls
  • D) To connect navigation components
Answer: A) To manage dynamic data within a component
Explanation: State is used to manage data that affects the rendering of a component.

2. What hook is used to create state in functional components?

  • A) useState
  • B) useEffect
  • C) useContext
  • D) useReducer
Answer: A) useState
Explanation: The useState hook is used to manage state in functional components.

3. What function is used to update state when using the useState hook?

  • A) setState
  • B) updateState
  • C) set
  • D) changeState
Answer: A) setState
Explanation: The setter function returned by useState (e.g., setState) is used to update the state value.

4. What does passing an empty array [] as a dependency to useEffect do?

  • A) Ensures the effect runs only once
  • B) Runs the effect on every render
  • C) Triggers the effect only when props change
  • D) Causes the component to re-render infinitely
Answer: A) Ensures the effect runs only once
Explanation: Passing an empty array makes the effect run only after the initial render.

5. How do you clean up side effects in useEffect?

  • A) Return a cleanup function
  • B) Call clean()
  • C) Use a try-finally block
  • D) Set the dependency array to null
Answer: A) Return a cleanup function
Explanation: To clean up side effects (e.g., subscriptions or timers), useEffect can return a cleanup function.

6. What is the purpose of the dependency array in useEffect?

  • A) To run the effect only when the specified values change
  • B) To trigger updates to props
  • C) To reset state after each render
  • D) To avoid re-renders
Answer: A) To run the effect only when the specified values change
Explanation: The dependency array determines when the effect should re-run, based on changes in the specified values.

7. What happens if you omit the dependency array in useEffect?

  • A) The effect runs on every render
  • B) The effect runs only once
  • C) The effect doesn't run at all
  • D) The component stops rendering
Answer: A) The effect runs on every render
Explanation: Omitting the dependency array causes useEffect to run after every render cycle.

8. What type of data structure does the useState hook return?

  • A) An object with state and setter functions
  • B) A single value
  • C) An array with the state variable and a function to update it
  • D) A string with state information
Answer: C) An array with the state variable and a function to update it
Explanation: useState returns a two-element array: the current state value and a function to update it.

9. How do you set the initial value of a state variable using useState?

  • A) Pass the value as an argument to useState()
  • B) Assign it directly to the variable
  • C) Use setState() immediately
  • D) Use a constructor function
Answer: A) Pass the value as an argument to useState()
Explanation: The initial value is provided as an argument to useState(), e.g., useState(0).

10. What should you avoid when directly updating a state variable?

  • A) Mutating the state object
  • B) Using the setState() function
  • C) Logging the state value
  • D) Passing a callback to update the state
Answer: A) Mutating the state object
Explanation: State should never be mutated directly. Always use the setter function returned by useState.

11. What happens if you call the state setter function with the same value as the current state?

  • A) The component does not re-render
  • B) The component re-renders unnecessarily
  • C) The state becomes undefined
  • D) An error is thrown
Answer: A) The component does not re-render
Explanation: React optimizes renders and avoids unnecessary re-renders if the state value does not change.

12. What lifecycle behavior does useEffect mimic when used with an empty dependency array?

  • A) componentDidMount
  • B) componentDidUpdate
  • C) componentWillUnmount
  • D) render
Answer: A) componentDidMount
Explanation: When useEffect is provided with an empty dependency array, it mimics the behavior of componentDidMount.

13. How do you clean up a timer set in useEffect?

  • A) Clear the timer in the cleanup function
  • B) Use setState(null)
  • C) Call useState() again
  • D) Return null from useEffect
Answer: A) Clear the timer in the cleanup function
Explanation: Cleanup functions in useEffect should clear timers using clearTimeout() or clearInterval().

14. What value does useState() return when no initial value is provided?

  • A) undefined
  • B) null
  • C) 0
  • D) An empty string
Answer: A) undefined
Explanation: If no initial value is provided, the state variable is initialized as undefined.

15. When should you avoid using the useEffect hook?

  • A) When no side effects are needed
  • B) When updating state
  • C) When fetching data
  • D) When adding event listeners
Answer: A) When no side effects are needed
Explanation: useEffect is only necessary when there are side effects to manage, such as data fetching or subscriptions.

16. What is the purpose of the cleanup function in useEffect?

  • A) To remove side effects before the component unmounts or re-runs
  • B) To reset the state of the component
  • C) To prevent state updates
  • D) To optimize re-renders
Answer: A) To remove side effects before the component unmounts or re-runs
Explanation: The cleanup function in useEffect removes side effects such as subscriptions, timers, or event listeners.

17. What happens if you provide incorrect dependencies in the useEffect dependency array?

  • A) The effect might not run as expected
  • B) The component will crash
  • C) The state becomes undefined
  • D) An error is thrown
Answer: A) The effect might not run as expected
Explanation: Incorrect dependencies can cause the effect to run at unexpected times or fail to respond to changes correctly.

18. Can you use multiple useEffect hooks in a single component?

  • A) Yes, each can handle a separate side effect
  • B) No, only one useEffect is allowed
  • C) Yes, but they must share the same dependencies
  • D) No, it will cause a conflict
Answer: A) Yes, each can handle a separate side effect
Explanation: You can use multiple useEffect hooks in a component to handle different side effects independently.

19. What does useEffect return?

  • A) A cleanup function
  • B) A value to update state
  • C) A JSX element
  • D) Nothing
Answer: D) Nothing
Explanation: useEffect itself does not return a value. However, it can optionally return a cleanup function.

20. What is the primary advantage of using functional components with hooks like useState and useEffect?

  • A) Simplifies state and lifecycle management
  • B) Improves render performance
  • C) Reduces the need for props
  • D) Eliminates re-renders
Answer: A) Simplifies state and lifecycle management
Explanation: Hooks simplify managing state and side effects in functional components without requiring class components.

21. Which of the following is true about the dependency array in useEffect?

  • A) It must always include all variables used in the effect
  • B) It is optional, but omitting it runs the effect after every render
  • C) It should always be empty
  • D) It cannot contain state variables
Answer: B) It is optional, but omitting it runs the effect after every render
Explanation: Omitting the dependency array makes the effect run after every render, while including variables ensures it runs only when those variables change.

22. What happens when setState is called in a functional component?

  • A) React schedules a re-render of the component
  • B) The entire application is reloaded
  • C) State variables are reset to their initial values
  • D) The DOM is updated directly
Answer: A) React schedules a re-render of the component
Explanation: Calling setState schedules a re-render of the component to reflect the updated state.

23. What should you include in the dependency array of useEffect?

  • A) All variables and props used inside the effect
  • B) Only variables that change state
  • C) Only global variables
  • D) Nothing
Answer: A) All variables and props used inside the effect
Explanation: To ensure correct behavior, include all variables and props referenced inside the effect in the dependency array.

24. What is the primary use case for useState?

  • A) Managing local state within a functional component
  • B) Sharing state across multiple components
  • C) Optimizing renders
  • D) Handling API requests
Answer: A) Managing local state within a functional component
Explanation: useState is designed to manage local component state in functional components.

25. What is the return value of the useEffect hook?

  • A) A cleanup function or nothing
  • B) A state variable
  • C) A new component
  • D) A JSX element
Answer: A) A cleanup function or nothing
Explanation: useEffect optionally returns a cleanup function; otherwise, it returns nothing.

26. What happens if you omit the second argument (dependency array) in useEffect?

  • A) The effect runs after every render
  • B) The effect runs only once
  • C) The effect doesn't run at all
  • D) The effect causes an error
Answer: A) The effect runs after every render
Explanation: Omitting the dependency array causes useEffect to run after every render cycle.

27. What is a common mistake when using the dependency array in useEffect?

  • A) Omitting variables that are used inside the effect
  • B) Including only state variables
  • C) Using an empty array
  • D) Providing the wrong data type
Answer: A) Omitting variables that are used inside the effect
Explanation: Failing to include variables referenced in the effect can lead to unexpected behavior.

28. How do you initialize state using the useState hook?

  • A) Pass an initial value to the useState function
  • B) Assign a value directly to the state variable
  • C) Use a constructor to set the state
  • D) Set the value in a lifecycle method
Answer: A) Pass an initial value to the useState function
Explanation: State is initialized by passing an initial value as an argument to useState, e.g., useState(0).

29. When does the cleanup function in useEffect get executed?

  • A) Before the component unmounts or the effect re-runs
  • B) Immediately after the component mounts
  • C) During every render
  • D) Only when the dependency array is empty
Answer: A) Before the component unmounts or the effect re-runs
Explanation: The cleanup function runs before the component unmounts or when the effect is about to re-run due to dependency changes.

30. What is a key advantage of using hooks like useState and useEffect?

  • A) Enables state and side effects in functional components
  • B) Improves app performance by default
  • C) Reduces the need for CSS styling
  • D) Makes React compatible with other libraries
Answer: A) Enables state and side effects in functional components
Explanation: Hooks like useState and useEffect allow functional components to manage state and lifecycle events.
