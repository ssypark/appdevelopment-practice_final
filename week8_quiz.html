Quiz: Week 8 - Navigation in React Native

1. Which library is used for navigation in React Native?

  • A) React Navigation
  • B) React Router
  • C) Redux Navigator
  • D) Expo Navigation
Answer: A) React Navigation
Explanation: React Navigation is the most widely used library for handling navigation in React Native apps.

2. What is the purpose of a NavigationContainer in React Navigation?

  • A) To manage app-wide state
  • B) To wrap all navigators in the app
  • C) To style the navigation bar
  • D) To handle animations
Answer: B) To wrap all navigators in the app
Explanation: NavigationContainer is a required component that encapsulates all navigators in a React Navigation app.

3. Which navigator allows switching between tabs?

  • A) Stack Navigator
  • B) Bottom Tab Navigator
  • C) Drawer Navigator
  • D) Switch Navigator
Answer: B) Bottom Tab Navigator
Explanation: Bottom Tab Navigator is used to create tab-based navigation at the bottom of the screen.

4. What is the initial screen in a navigator called?

  • A) Home Screen
  • B) Default Screen
  • C) Root Screen
  • D) Initial Route
Answer: D) Initial Route
Explanation: The initial screen of a navigator is defined using the initialRouteName property.

5. Which property sets a custom title for a screen?

  • A) headerTitle
  • B) title
  • C) screenName
  • D) screenTitle
Answer: A) headerTitle
Explanation: The headerTitle property customizes the title displayed in the screen's header.

6. Which navigator is used for a swipeable side menu?

  • A) Drawer Navigator
  • B) Bottom Tab Navigator
  • C) Stack Navigator
  • D) Side Menu Navigator
Answer: A) Drawer Navigator
Explanation: Drawer Navigator is used to implement a swipeable or toggleable side menu.

7. What command is used to install the React Navigation library?

  • A) npm install react-navigation
  • B) npm install @react-navigation/native
  • C) expo install react-navigation
  • D) npm install react-navigation/core
Answer: B) npm install @react-navigation/native
Explanation: The correct command for installing React Navigation is npm install @react-navigation/native.

8. Which additional libraries are required for React Navigation?

  • A) react-native-screens and react-native-safe-area-context
  • B) react-native-router and react-native-navigation
  • C) react-native-core and react-navigation-stack
  • D) expo-router and expo-navigation
Answer: A) react-native-screens and react-native-safe-area-context
Explanation: React Navigation requires additional dependencies like react-native-screens and react-native-safe-area-context.

9. How do you create a stack navigator?

  • A) import { createStackNavigator } from '@react-navigation/native-stack';
  • B) import { createNavigator } from '@react-navigation/stack';
  • C) import { createStackNav } from 'react-native-navigation';
  • D) import { StackNavigator } from 'expo-navigation';
Answer: A) import { createStackNavigator } from '@react-navigation/native-stack';
Explanation: Use createStackNavigator from @react-navigation/native-stack to create stack-based navigation.

10. What method navigates to another screen in React Navigation?

  • A) navigate()
  • B) goTo()
  • C) routeTo()
  • D) switchScreen()
Answer: A) navigate()
Explanation: The navigate() method is used to switch to another screen in React Navigation.

11. What is the purpose of the headerShown option in screen configuration?

  • A) To toggle the visibility of the screen's header
  • B) To change the header's color
  • C) To set the title of the header
  • D) To align the header text
Answer: A) To toggle the visibility of the screen's header
Explanation: The headerShown option determines whether the header is displayed for a screen.

12. What is the main difference between stack and tab navigators?

  • A) Stack navigators are vertical, tabs are horizontal
  • B) Stack navigators have headers, tabs do not
  • C) Stack navigators move between screens linearly, tabs allow switching between screens directly
  • D) Stack navigators use swipe gestures, tabs do not
Answer: C) Stack navigators move between screens linearly, tabs allow switching between screens directly
Explanation: Stack navigators maintain a history of screens, while tabs switch between screens without a history.

13. How do you pass parameters to a screen?

  • A) Using navigation.setParams()
  • B) By including parameters in navigation.navigate()
  • C) Adding parameters to the screen's options
  • D) Using global state
Answer: B) By including parameters in navigation.navigate()
Explanation: Parameters can be passed by including them in the navigate() method, such as navigation.navigate('ScreenName', { paramKey: value }).

14. Which hook retrieves route parameters in a screen?

  • A) useParams()
  • B) useNavigationParams()
  • C) useRoute()
  • D) useScreenParams()
Answer: C) useRoute()
Explanation: The useRoute() hook retrieves route parameters passed to a screen.

15. What component is required to group multiple navigators?

  • A) NavigationContainer
  • B) NavigatorGroup
  • C) RootNavigator
  • D) AppContainer
Answer: A) NavigationContainer
Explanation: The NavigationContainer component is required to group multiple navigators in React Navigation.

16. What does the screenOptions property in a navigator configure?

  • A) Global styles and behaviors for all screens in the navigator
  • B) Animation settings for transitions
  • C) NavigationContainer themes
  • D) Gesture handling for a specific screen
Answer: A) Global styles and behaviors for all screens in the navigator
Explanation: The screenOptions property applies styles or configurations globally to all screens in a navigator.

17. Which navigator supports modals by default?

  • A) Drawer Navigator
  • B) Stack Navigator
  • C) Bottom Tab Navigator
  • D) Modal Navigator
Answer: B) Stack Navigator
Explanation: Stack Navigator can display modals by specifying the mode: 'modal' option.

18. How can you hide the tab bar for a specific screen in a Bottom Tab Navigator?

  • A) Set tabBarVisible: false in screenOptions
  • B) Use the hideTabBar function
  • C) Add display: none in CSS
  • D) Set tabBarStyle: { display: 'none' } in options
Answer: D) Set tabBarStyle: { display: 'none' } in options
Explanation: To hide the tab bar for a specific screen, use tabBarStyle with the display: 'none' property.

19. What is the primary purpose of react-native-screens in navigation?

  • A) To manage animations for transitions
  • B) To improve performance by using native screen management
  • C) To add default header styles
  • D) To enable gesture handling
Answer: B) To improve performance by using native screen management
Explanation: The react-native-screens library enhances performance by managing screens natively.

20. What is the difference between replace() and navigate() in React Navigation?

  • A) replace() removes the previous route, while navigate() adds to the stack
  • B) replace() is used for tabs, while navigate() is for stacks
  • C) replace() requires parameters, navigate() does not
  • D) Both perform the same function
Answer: A) replace() removes the previous route, while navigate() adds to the stack
Explanation: The replace() method replaces the current screen in the stack without adding a new one.

21. How do you enable gestures for a screen?

  • A) Use gestureEnabled: true in screenOptions
  • B) Add the enableGestures prop
  • C) Configure gestures in NavigationContainer
  • D) Use swipeEnabled: true in styles
Answer: A) Use gestureEnabled: true in screenOptions
Explanation: Gestures can be enabled for a screen using the gestureEnabled option in screenOptions.

22. Which hook provides access to the navigation object?

  • A) useNavigation()
  • B) useRoute()
  • C) useNavigate()
  • D) useHistory()
Answer: A) useNavigation()
Explanation: The useNavigation() hook allows access to the navigation object in functional components.

23. How do you add a custom component to a header?

  • A) Use headerTitle: () => <CustomComponent />
  • B) Set customHeader in screenOptions
  • C) Pass the component to NavigationContainer
  • D) Modify headerStyle
Answer: A) Use headerTitle: () => <CustomComponent />
Explanation: A custom component can be added to the header by setting headerTitle to a function that returns the component.

24. What property changes the background color of the header?

  • A) headerStyle
  • B) backgroundColor
  • C) headerBackground
  • D) themeColor
Answer: A) headerStyle
Explanation: The headerStyle property is used to apply styles, including background color, to the header.

25. Which navigator is most suitable for a master-detail layout?

  • A) Stack Navigator
  • B) Drawer Navigator
  • C) Bottom Tab Navigator
  • D) Nested Navigators
Answer: A) Stack Navigator
Explanation: Stack Navigator is commonly used for master-detail layouts due to its ability to push and pop screens.

26. What is the purpose of nested navigators in React Navigation?

  • B) To share navigation parameters between apps
  • C) To add gestures to all screens
  • D) To apply global screen styles
Answer: A) To create hierarchical navigation structures
Explanation: Nested navigators allow the creation of complex, hierarchical navigation patterns.

27. What is the difference between push() and navigate() in React Navigation?

  • A) push() adds a duplicate screen to the stack, navigate() does not
  • B) push() is for tabs, navigate() is for stacks
  • C) push() requires parameters, navigate() does not
  • D) Both perform the same function
Answer: A) push() adds a duplicate screen to the stack, navigate() does not
Explanation: The push() method adds a new instance of a screen, while navigate() navigates to an existing instance if itâ€™s already in the stack.

  • D) Use tabBarStyle in the navigator's options

  • B) The header will be hidden globally
  • C) The header will display an error message
  • D) The screen will not load
